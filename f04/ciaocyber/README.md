# CIAO_CYBER

### Sobre:
O desafio pedia a criação de uma página HTML simples para praticar o uso de **Bind Mount** com `Docker` e `Nginx`.

---
#### O que aprendi:

* Criar e servir um site estático utilizando o Nginx em um container
* Utilizar Bind Mount para espelhar uma pasta do host dentro do container
* Editar arquivos localmente e visualizar as alterações em tempo real, sem necessidade de reconstruir a imagem
* Entender a diferença entre **Bind Mount** e **Volumes Nomeados**
* Aplicar o conceito de Read-only (`:ro`), permitindo que o container apenas leia os arquivos
* Inspecionar containers utilizando `docker inspect`, `docker exec` e `docker logs`.
* Diagnosticar problemas comuns relacionados a portas, mounts e estrutura de diretórios.
* Reforçar conhecimentos básicos de HTML, CSS e Figma durante a criação de uma página personalizada.

---
Embora o objetivo técnico do exercicio fosse o Docker, me empolguei na ideia de aprimorar e tornar mais completo trazendo mais personalidade.

Durante o desenvolvimento utilizei ajuda da IA para acelerar e criar o corpo do projeto deixando-o mais completo.

Aproveitei este projeto para relembrar conhecimentos de HTML,CSS e Figma Design que havia adquirido anteriormente em cursos na qual não cheguei a concluir devido a problemas pessoais, com a memória ainda fresca utilizei o desafio como uma oportunidade prática de aprendizado.

Participei ativamente do processo realizando adaptações necessárias, ajuste na estrutura, modificações no layout, testes e correções de acordo com o meu gosto pessoal.

---

### Desafios encontrados:

* Compreender como o Bind Mount substitui o conteúdo existente dentro do container.
* Entender por que alterações feitas diretamente no container não são persistidas
* Resolver conflitos de portas quando outro container já estava utilizando a porta `8080`.
* Organizar a estrutura do site para deixar mais agradável visualmente.
* Equilibrar o uso da IA com os estudos sem depender 100% dele.
